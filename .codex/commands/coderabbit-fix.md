# coderabbit fix

CodeRabbit のレビュー指摘を取得し、必要な修正を実装する。

## 手順

1. PR 番号が未指定なら対象 PR を確認する。
2. GitHub コネクタまたは `gh api` で CodeRabbit のレビュー、インラインコメント、Issue コメントを取得する。
3. 指摘を分類する。
   - 必須対応: バグ、セキュリティ、型エラー、ルール違反
   - 推奨対応: 可読性、性能、ベストプラクティス
   - 任意対応: スタイル、命名、軽微な提案
4. 対応方針をユーザーに確認する。既定は必須対応 + 推奨対応。
5. 対象ファイルを読み、テストが必要な場合は先に追加する。
6. 修正後に存在する品質チェックを実行する。
7. ユーザーが希望する場合は smart commit と push を行う。

## Codex 注意点

- CodeRabbit の提案を鵜呑みにせず、このリポジトリのルールと照合する。
- 既存のユーザー変更は巻き戻さない。
- Claude 版の詳細テンプレートが必要な場合は `.claude/commands/coderabbit-fix.md` も参照する。
