# review PR

Pull Request をコードレビューする。

## 手順

1. PR 番号または URL が未指定なら、候補を取得して対象を確認する。
2. GitHub コネクタが使える場合はそれを優先し、なければ `gh pr view` / `gh pr diff` を使う。
3. 差分、コミット、チェック状態を確認する。
4. Evaluator フェーズとして、以下の観点で独立レビューする。
   - 正確性
   - 型安全
   - セキュリティ
   - パフォーマンス
   - テスト
   - 規約遵守
5. 指摘は重要度順に、ファイルと行番号つきで報告する。
6. ユーザーが投稿まで依頼している場合のみ、GitHub にレビューを投稿する。

## 報告形式

```md
## レビュー結果: PR #<番号>

### 判定: LGTM / 要修正 / 差し戻し

### 指摘事項
- [重要度: 高/中/低] path/to/file.ts:行番号 — 指摘内容

### 残リスク
- あれば記載
```

## Codex 注意点

- レビューでは要約より指摘を先に出す。
- 投稿前に、コメント対象が自分の PR か他人の PR かを確認する。
- Claude 版の詳細テンプレートが必要な場合は `.claude/commands/review-pr.md` も参照する。
