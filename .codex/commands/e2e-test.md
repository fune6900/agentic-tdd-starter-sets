# e2e test

E2E 観点で主要フローを検証する。

## 手順

1. 対象 URL またはフローが未指定なら `http://localhost:3000` を既定にする。
2. 開発サーバーが必要なら起動状態を確認する。
3. Codex のブラウザ操作ツールが使える場合はそれを優先する。
4. ブラウザ操作ツールが未ロードの場合は `tool_search` で browser / playwright を探す。
5. 主要フロー、エンプティステート、エラーステート、バリデーション、二重送信防止を確認する。
6. 既存の `npm run e2e` があれば実行する。
7. 新しい回帰観点が見つかった場合は、ユーザーの依頼範囲に応じて Playwright テストを追加する。

## 報告形式

```md
## E2E テスト結果

### 総合判定: 合格 / 要修正 / 実行不能
対象URL: <url>

### 確認したフロー
- <flow>: PASS / FAIL

### 問題
- あれば記載
```

## Codex 注意点

- Claude 版 MCP 名は `.codex/mcp-map.md` の対応表に従って読み替える。
- Claude 版の詳細テンプレートが必要な場合は `.claude/commands/e2e-test.md` も参照する。
