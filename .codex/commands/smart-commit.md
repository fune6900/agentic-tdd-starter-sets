# smart commit

変更内容を確認し、品質チェックを通ったファイルだけを意味のある単位でコミットする。

## 手順

1. `git status --short` と `git diff` で変更範囲を確認する。
2. `.env`、秘密鍵、トークン、`*secret*` などの機密ファイルが含まれていないか確認する。
3. 存在するチェックだけを実行する。
   - `npm test -- --run`
   - `npm run typecheck`
   - `npm run lint`
   - `npm run build`
4. 失敗したチェックがあればコミットせず、失敗内容を報告する。
5. 無関係な変更が混在している場合は、ユーザーに分割方針を確認する。
6. コミットメッセージが指定されていなければ、差分から英語の Conventional Commit を作る。
7. 対象ファイルをステージしてコミットする。

## Codex 注意点

- 既存のユーザー変更は巻き戻さない。
- push は明示指示がある場合のみ行う。
- コミット成功後はコミットハッシュとメッセージを報告する。
- Claude 版の詳細テンプレートが必要な場合は `.claude/commands/smart-commit.md` も参照する。
