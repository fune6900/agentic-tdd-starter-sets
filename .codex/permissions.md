# Codex 権限ガイド

Claude の `.claude/settings.json` の permissions は Codex にはそのまま適用されない。
Codex では、実行環境のサンドボックス、承認ポリシー、この文書の運用ルールに従う。

## 既定で許可してよい操作

- リポジトリ内ファイルの読み取り
- ユーザー依頼に必要な範囲のリポジトリ内ファイル編集
- `git status`、`git diff`、`git log`、`git branch --show-current`
- 定義済み npm scripts の実行
- `rg`、`find`、`ls` などの調査コマンド

## 事前確認または明示指示が必要な操作

- コミット作成
- push
- PR 作成
- PR マージ
- リモートブランチ削除
- 外部ネットワークアクセスが必要な調査
- リポジトリ外への書き込み

## 禁止または強い制限がある操作

- `git reset --hard`
- `git clean -fd`
- force push
- rebase
- `sudo`
- `rm -rf` など広範囲削除
- `.env`、秘密鍵、トークン、`*secret*` の読み書き
- `--no-verify` によるチェック回避

ユーザーが明示した場合でも、破壊的操作は対象、影響、代替案を確認してから実行する。

## Codex での実務ルール

- 既存のユーザー変更は巻き戻さない。
- 無関係なファイルの変更は触らない。
- 権限が足りない場合は、必要最小限のパスまたはネットワークだけを要求する。
- シークレット値は出力しない。
