# Codex 品質ゲート

完了前に、対象プロジェクトで定義されているチェックだけを実行する。

## 標準チェック順

```bash
npm test -- --run
npm run typecheck
npm run lint
npm run build
```

## 実行ルール

- `package.json` が存在しない場合は、npm チェックは実行不能として報告する。
- script が定義されていないコマンドはスキップし、最終報告で明示する。
- 依存関係が未インストールの場合は、勝手にインストールせず状況を報告する。
- チェックが失敗した場合は、可能な範囲で修正して再実行する。
- 修正できない場合は、失敗したコマンドと要点を報告する。

## TDD ルール

- 機能追加とバグ修正は、原則としてテストを先に追加または更新する。
- ドキュメント、設定文言、翻訳のみの変更では、TDD は不要。構文チェックや差分確認を優先する。
- UI 変更では、可能な限りブラウザ確認またはスクリーンショット確認を行う。

## 完了報告に含めるもの

- 変更した主なファイル
- 実行したチェック
- スキップしたチェックと理由
- 残リスクや未確認事項
