lint・typecheck を通過したファイルのみをステージし、意味のあるコミットメッセージで自動コミットする。

## 対象
- 引数 $ARGUMENTS にコミットメッセージを指定できる。未指定の場合は差分から自動生成する。

## 手順

### Phase 1: 品質チェック
1. `npm run lint` を実行する。エラーがあれば即座に報告してコミットを中止する。
2. `npm run typecheck` を実行する。型エラーがあれば即座に報告してコミットを中止する。
3. `npm test -- --run` を実行する（設定済みの場合）。テストが落ちていればコミットを中止する。

### Phase 2: 差分確認
1. `git status` で変更ファイルを確認する。
2. `git diff` で変更内容を確認する。
3. `.env` / `*.key` / `secrets.*` などの機密ファイルが含まれていないことを確認する。含まれていれば除外してマスターに警告する。

### Phase 3: コミットメッセージ生成
- $ARGUMENTS が指定されている場合: そのメッセージをそのまま使用する。
- 未指定の場合: 差分を分析し、以下のフォーマットでメッセージを生成する。

```
<種別>: <変更の理由または内容（英語・50文字以内）>

種別: feat / fix / refactor / test / chore / ci
```

### Phase 4: ステージ & コミット
1. `git add` で変更ファイルをステージする（機密ファイルを除く）。
2. 以下でコミットする:

```bash
git commit -m "$(cat <<'EOF'
<生成したメッセージ>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

3. コミット成功後、コミットハッシュとメッセージを報告する。

## 注意
- lint / typecheck / test のいずれかが失敗した場合、コミットは実行しない。エラー内容を報告し、修正を促すこと。
- コミットは push しない。push は明示的に指示された場合のみ行う。
- 1コミット = 1つの意味のある変更単位を守ること。無関係な変更が混在している場合は分割を提案する。
