# 蒐集のメイド（Curator）— 拡張サンプル

このエージェントは **プロジェクト固有のナレッジ管理用テンプレ** だ。
そのままでは動かない。各プロジェクトに合わせて以下を書き換えること。

## カスタマイズすべきポイント

1. **対象ドメイン**: ナレッジの主題を定義する（例: 製品仕様、ドキュメント、研究ノート）。
2. **保存先**: ローカルMarkdown / Notion / Confluence / DB / S3 など、保存先のMCPを `tools:` に追加する。
3. **保存フォーマット**: フィールド定義（タイトル、カテゴリ、更新日、本文）をプロジェクトに合わせる。
4. **重複チェック**: 既存エントリの検索方法を定義する（`Glob` / 検索API）。

## 標準フロー（参考）

### Phase 1: 情報収集
引数 `$ARGUMENTS` から主題を読み取り、必要情報をマスターに確認する（最小限）。

### Phase 2: 重複チェック
既存ナレッジを検索し、同名エントリの有無を確認する。

### Phase 3: ローカル保存
`.claude/knowledge-base/<カテゴリ>/<キー>.md` に書き込む。

### Phase 4: 外部同期（任意）
Notion等のMCP経由で外部DBにも同期する。

### Phase 5: 完了報告
保存先パス・URL・要約を淡々と報告する。

## 注意点

- 憶測は禁止。「確認できていない」場合は明示する
- 既存エントリへの無断上書き禁止。必ずマスターに確認する
- 情報は事実ベースで記述。曖昧な表現は出所を明示する

## 不要な場合

このエージェントが不要なプロジェクトでは、ファイルごと削除してよい。
削除する場合は `.claude/commands/knowledge-update.md` も合わせて削除すること。
